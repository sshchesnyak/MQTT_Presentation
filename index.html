Структура IoT-
платформы

  • Mote – оконечное устройство, КГ
  • Base Station – базовая станция
  • Network Server (NS) - для настройки параметров
    радиопередачи по каналам Ethernet и LoRaWAN и
    аутентификации пакетов
  • Application Server (AS) - для шифрования и
    расшифрования пакетов
  • Customer Server (CS) - для парсинга данных в целях
    объединения в структуры для обработки, хранения
    и передачи

Выбор протокола

  • Критерии:
    • Надежность
    • Энергоэффективность
    • Защищенность
    • Скорость реакции на события
    • Возможность группового управления
    • Простота реализации
  • CoAP – ненадежен и не энергоэффективен, медленно реагирует на
    события
  • HTTP – ненадежен и не энергоэффективен, нет группового управления
  • MQTT – оптимальное решение

MQTT

  • MQTT – Message Queueing Telemetry Transport
  • Предназначен для сетей с ограниченными ресурсами
    и пропускной способностью
  • 1999 г, создан для нефтегазовой промышленности
